<decisions_made>

- Grace logic lives in GameLoop (renderer layer), not engine — RaceController only knows one car
- Discriminated union `VsAiGraceState` with `'countdown' | 'resolved'` status (not flat boolean interface)
- Both `humanTotalTicks` and `aiTotalTicks` are `number | null` (human can DNF too)
- Extract to `private updateGrace()` method (not a separate GraceCoordinator class — less file churn)
- Extract `private shouldStepAi()` for readability
- `GRACE_PERIOD_TICKS` is module-private, derived from `FIXED_DT_MS` (not magic number 300)
- Use BitmapText (not Text) for countdown timer — avoids 60 canvas rasterizations/sec
- Use `scale.x` for progress bar (not `clear()/redraw()` — PixiJS memory leak risk)
- Use `.tint` on white BitmapFont for color interpolation (zero cost)
- Keep progress bar (game UX research confirms it's more scannable than digits during racing)
- Grace state persists as `resolved` (not nulled) until `resetWorld()` — Finished overlay needs it
- AI finish detection derived from grace state existence (no separate one-shot boolean flag)
- Init grace BETWEEN human step and AI gate (fixes same-tick ordering bug)
</decisions_made>
