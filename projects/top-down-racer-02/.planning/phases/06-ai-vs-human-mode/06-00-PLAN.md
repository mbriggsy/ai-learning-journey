<objective>
Create the Python ONNX export script that converts a trained SB3 PPO model to ONNX format and dumps VecNormalize stats to JSON. This plan runs at Wave 0 — it has zero TypeScript coupling and can execute independently before any other Phase 6 work.

Purpose: Separating the Python export script into its own plan keeps Plan 01 focused on TypeScript infrastructure and avoids coupling a Python-only task to the TypeScript TDD pipeline.

Output: `python/training/export_onnx.py` — a standalone script the user runs once when a trained model is ready. Also updates `python/requirements.txt` and `.gitignore`.

### Research Insights

**Script Location:** Place at `python/training/export_onnx.py` (not `python/export_onnx.py`). All existing Python scripts live under `python/training/` — train_ppo.py, train_sac.py, evaluate.py, benchmark.py, callbacks.py. Following this convention ensures `python -m training.export_onnx` works and keeps the Python package structure consistent.

**Model Size Context:** The PPO model has ~5,315 parameters producing an ONNX file of ~25-30KB. This is tiny — no quantization or optimization needed. Sub-millisecond inference on WASM backend.

**Security:** `pickle.load()` on untrusted files enables arbitrary code execution (CWE-502). Since VecNormalize.load() handles the same `.pkl` files safely via SB3's internal loading, the pickle fallback is unnecessary risk with zero benefit. Hard fail if VecNormalize.load() fails — the user can investigate the specific error.
</objective>

  <action>
    Create python/training/export_onnx.py — a one-time script to export the trained PPO model to ONNX format and dump VecNormalize stats to JSON.

    **Location:** `python/training/export_onnx.py` (NOT `python/export_onnx.py`). This matches the existing convention where all Python scripts live under `python/training/`.

    **Invocation:** `python -m training.export_onnx --model PATH --vecnorm PATH` (from the `python/` directory, inside the venv).

    The script must:
    1. Accept CLI args via argparse:
       - `--model` (REQUIRED, no default — different users have different checkpoint paths)
       - `--vecnorm` (REQUIRED, no default — same reason)
       - `--output-dir` (optional, default: `../public/assets/` relative to script)
    2. Create the output directory if it doesn't exist (using pathlib)
    3. Load model with `PPO.load(model_path, device='cpu')`
    4. Derive `obs_size` from `model.observation_space.shape[0]` (NOT hardcoded 14)
    5. Wrap policy in `OnnxableSB3Policy(th.nn.Module)` that calls `policy(obs, deterministic=True)`
    6. Call `onnx_policy.eval()` before export
    7. Export inside `torch.no_grad()` context manager
    8. Export with `torch.onnx.export`:
       - `opset_version=17`
       - `input_names=["obs"]`
       - `output_names=["actions"]` (ONLY actions — values/log_probs are unused by the browser)
       - `dynamic_axes={"obs": {0: "batch_size"}, "actions": {0: "batch_size"}}`
       - `dynamo=False` (explicit — PyTorch 2.9+ changed the default to True, which can break SB3 policy wrappers)
    9. Load VecNormalize stats using `VecNormalize.load(vecnorm_path, venv=None)`:
       - Access `.obs_rms.mean`, `.obs_rms.var`, `.clip_obs`, `.epsilon`
       - NO pickle.load() fallback (security risk: CWE-502 arbitrary code execution)
       - If VecNormalize.load() fails, print the error and exit with code 1
    10. Write output_dir/vecnorm_stats.json with:
        ```json
        { "obs_mean": [...], "obs_var": [...], "clip_obs": 10.0, "epsilon": 1e-8 }
        ```
        Convert numpy arrays to Python lists via `.tolist()`.
    11. Verify exported model by loading it back with `ort.InferenceSession` and running dummy inference:
        - Print input names, output names, and output shape
        - Use dummy input `np.random.randn(1, obs_size).astype(np.float32)`
        - Verify the "actions" output has shape `(1, 3)` — [steer, throttle, brake]
    12. Print success summary with file sizes (human-readable)

    **Use pathlib throughout** (matches project convention — no `os.path`):
    ```python
    from pathlib import Path

    output_dir = Path(args.output_dir)
    output_dir.mkdir(parents=True, exist_ok=True)
    model_output = output_dir / "model.onnx"
    stats_output = output_dir / "vecnorm_stats.json"
    ```

    **OnnxableSB3Policy with type hints:**
    ```python
    class OnnxableSB3Policy(th.nn.Module):
        """Wrapper that extracts only actions from SB3 policy forward pass."""

        def __init__(self, policy: BasePolicy) -> None:
            super().__init__()
            self.policy = policy

        def forward(self, obs: th.Tensor) -> th.Tensor:
            actions, _values, _log_probs = self.policy(obs, deterministic=True)
            return actions  # Only export actions — browser only needs [steer, throttle, brake]
    ```
    Note: OnnxableSB3Policy.forward() returns ONLY actions (not the full tuple). This ensures `torch.onnx.export` with `output_names=["actions"]` produces a clean graph with a single output node.

    **torch.onnx.export call:**
    ```python
    dummy_input = th.randn(1, obs_size)
    onnx_policy.eval()
    with th.no_grad():
        th.onnx.export(
            onnx_policy,
            dummy_input,
            str(model_output),
            opset_version=17,
            input_names=["obs"],
            output_names=["actions"],
            dynamic_axes={"obs": {0: "batch_size"}, "actions": {0: "batch_size"}},
            dynamo=False,  # Explicit: PyTorch 2.9+ defaults to True, breaks SB3 wrappers
        )
    ```

    **VecNormalize.load() — no pickle fallback:**
    ```python
    from stable_baselines3.common.vec_env import VecNormalize

    def export_vecnorm(vecnorm_path: Path, output_path: Path) -> None:
        # VecNormalize.load() handles .pkl safely via SB3 internals.
        # NO pickle.load() fallback — that's CWE-502 (arbitrary code execution).
        vn = VecNormalize.load(str(vecnorm_path), venv=None)
        stats = {
            "obs_mean": vn.obs_rms.mean.tolist(),
            "obs_var": vn.obs_rms.var.tolist(),
            "clip_obs": float(vn.clip_obs),
            "epsilon": float(vn.epsilon),
        }
        output_path.write_text(json.dumps(stats, indent=2))
        print(f"  VecNormalize stats: {output_path} ({output_path.stat().st_size} bytes)")
    ```

    **Docstring convention** (matches existing scripts like train_ppo.py, evaluate.py):
    ```python
    """Export trained SB3 PPO model to ONNX for browser inference.

    Usage:
        cd python
        python -m training.export_onnx --model models/your_model.zip --vecnorm models/your_vecnorm.pkl

    Requires: onnx, onnxruntime (see requirements.txt)
    """
    ```

    ### Edge Cases to Handle

    - **Model file doesn't exist:** Check with `Path.exists()` before loading, print clear error
    - **VecNormalize file doesn't exist:** Same check, clear error
    - **Output directory on different drive:** `mkdir(parents=True)` handles this
    - **obs_size mismatch:** If model.observation_space.shape[0] != len(vn.obs_rms.mean), warn
    - **Verification failure:** If ONNX output shape != (1, 3), print warning but don't fail (model architecture may vary)

    ### What NOT to Do (YAGNI)

    - Do NOT export "values" or "log_probs" outputs — BrowserAIRunner only reads "actions"
    - Do NOT add INT8 quantization — model is ~25-30KB, sub-ms inference already
    - Do NOT add batch inference support — browser runs single-sample inference
    - Do NOT add `--opset` CLI arg — opset 17 is the only tested version
    - Do NOT use `onnx.optimizer` — ONNX Runtime handles graph optimization at load time
