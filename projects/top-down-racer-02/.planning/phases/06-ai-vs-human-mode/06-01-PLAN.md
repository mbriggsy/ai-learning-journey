    <name>normalizeObservation</name>
    <files>src/ai/vecnormalize.ts, tests/ai/vecnormalize.test.ts</files>
    <behavior>
      Applies the SB3 VecNormalize formula exactly: clip((obs[i] - mean[i]) / sqrt(var[i] + epsilon), -clip_obs, clip_obs)

      Cases:
      - Zero mean, unit var, no clipping → obs returned unchanged (within float precision)
      - Value well above clip_obs → clipped to clip_obs
      - Value well below -clip_obs → clipped to -clip_obs
      - Different mean/var per element → each normalized independently
      - epsilon=1e-8, clip_obs=10.0 defaults match SB3 defaults
      - Output array has same length as input obs array (14 elements for full obs vector)
      - Dimension mismatch (obs.length !== mean.length) → throws descriptive Error
    </behavior>
    <implementation>
      Create src/ai/vecnormalize.ts:
      ```typescript
      export interface VecNormStats {
        readonly obsMean: number[];
        readonly obsVar: number[];
        readonly clipObs: number;
        readonly epsilon: number;
      }

      export function normalizeObservation(obs: readonly number[], stats: VecNormStats): number[] {
        const { obsMean, obsVar, clipObs, epsilon } = stats;

        if (obs.length !== obsMean.length || obs.length !== obsVar.length) {
          throw new Error(
            `VecNormalize dimension mismatch: obs=${obs.length}, mean=${obsMean.length}, var=${obsVar.length}`
          );
        }

        return obs.map((v, i) => {
          const normalized = (v - obsMean[i]) / Math.sqrt(obsVar[i] + epsilon);
          return Math.max(-clipObs, Math.min(clipObs, normalized));
        });
      }
      ```

      IMPORTANT — Naming convention: The TypeScript interface MUST use camelCase property names
      (`obsMean`, `obsVar`, `clipObs`) to match the codebase convention. Every existing interface
      (CarState, RewardConfig, EpisodeConfig, TrackState) uses camelCase without exception. The
      JSON file exported from Python uses snake_case; the deserialization point (BrowserAIRunner
      in Plan 02) remaps keys at load time:
      ```typescript

      const stats: VecNormStats = {
        obsMean: raw.obs_mean,
        obsVar: raw.obs_var,
        clipObs: raw.clip_obs,
        epsilon: raw.epsilon,
      };
      ```

      Properties are `readonly` to match the engine types convention (e.g., TrackState fields
      at src/engine/types.ts lines 107-126). Stats are loaded once from JSON and never mutated.

      - setHumanBest with NaN/Infinity/negative/zero → returns false (input rejected)

    <implementation>
      Create src/renderer/Leaderboard.ts:
      ```typescript
      const LEADERBOARD_KEY = 'tdr-leaderboard-v1';

      export interface TrackBests { human: number | null; ai: number | null; }
      export interface LeaderboardData { version: 1; tracks: Record<string, TrackBests>; }

      function empty(): LeaderboardData {
        return { version: 1, tracks: {} };
      }

      function isValidTicks(v: unknown): v is number {
        return typeof v === 'number' && Number.isFinite(v) && v > 0;
      }

      function load(): LeaderboardData {
        try {
          const raw = localStorage.getItem(LEADERBOARD_KEY);
          if (!raw) return empty();
          const parsed: unknown = JSON.parse(raw);
          if (typeof parsed !== 'object' || parsed === null || Array.isArray(parsed)) return empty();
          if ((parsed as any).version !== 1) return empty();
          if (typeof (parsed as any).tracks !== 'object') return empty();
          return parsed as LeaderboardData;
        } catch {
          return empty();
        }
      }

      function save(data: LeaderboardData): void {
        try {
          localStorage.setItem(LEADERBOARD_KEY, JSON.stringify(data));
        } catch {
          // QuotaExceededError, SecurityError — fail silently
        }
      }

      export function getLeaderboard(trackId: string): TrackBests {
        const data = load();
        const entry = data.tracks[trackId];
        return {
          human: entry?.human ?? null,
          ai: entry?.ai ?? null,
        };
      }

      export function setHumanBest(trackId: string, ticks: number): boolean {
        if (!isValidTicks(ticks)) return false;
        const data = load();
        const entry = data.tracks[trackId] ?? { human: null, ai: null };
        if (entry.human !== null && ticks >= entry.human) return false;
        entry.human = ticks;
        data.tracks[trackId] = entry;
        save(data);
        return true;
      }

      export function setAiBest(trackId: string, ticks: number): boolean {
        if (!isValidTicks(ticks)) return false;
        const data = load();
        const entry = data.tracks[trackId] ?? { human: null, ai: null };
        if (entry.ai !== null && ticks >= entry.ai) return false;
        entry.ai = ticks;
        data.tracks[trackId] = entry;
        save(data);
        return true;
      }
      ```

      IMPORTANT — Input validation: Use `Number.isFinite(ticks) && ticks > 0` (via `isValidTicks`)
      rather than just `ticks > 0`. This prevents Infinity from being treated as a valid best time
      (`Infinity > 0` is true). JSON.stringify serializes both NaN and Infinity as `null`, which
      would silently corrupt the "no record" vs "had a record" distinction.

      IMPORTANT — Nullish coalescing: Use `entry?.human ?? null` in getLeaderboard(), not just
      `entry.human`. JSON.stringify drops `undefined` values, so after a round-trip, a key might
      be missing entirely. The `?? null` ensures consistent return type.

      IMPORTANT — "Only write if better" semantics: Both setHumanBest and setAiBest must check
      `ticks >= current` (not just `ticks > current`). This preserves the existing BestTimes.ts
      contract that ScreenManager.ts (line 173) relies on.
