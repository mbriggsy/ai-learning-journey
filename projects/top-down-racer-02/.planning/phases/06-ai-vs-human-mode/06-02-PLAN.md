  **Vercel Deployment:**
  - Vercel serves `.wasm` files with correct `Content-Type: application/wasm` by default — no vercel.json needed for MIME types

      /**
       * BrowserAIRunner — Browser-side ONNX inference for the AI ghost car.
       *
       * Loads a trained PPO model (exported from SB3 via torch.onnx.export) and
       * runs per-tick inference using onnxruntime-web's WASM backend.
       * This module is browser-only — not used by the Python training pipeline.
       */
      import * as ort from 'onnxruntime-web';
      import { normalizeObservation, type VecNormStats } from './vecnormalize';
      import { OBSERVATION_SIZE } from './observations';

      // ──────────────────────────────────────────────────────────
      // WASM Configuration (must be set BEFORE any InferenceSession.create)
      // ──────────────────────────────────────────────────────────

      if (typeof ort.env !== 'undefined' && ort.env.wasm) {
        ort.env.wasm.wasmPaths = '/assets/ort/';
        ort.env.wasm.numThreads = 1; // Optimal for tiny MLP; avoids crossOriginIsolated requirement
      }

      // ──────────────────────────────────────────────────────────
      // Runtime Validation
      // ──────────────────────────────────────────────────────────

      /** Type guard for validating VecNormStats JSON from network. */
      function isValidStatsJson(value: unknown): value is {
        obs_mean: number[];
        obs_var: number[];
        clip_obs: number;
        epsilon: number;
      } {
        if (typeof value !== 'object' || value === null) return false;
        const obj = value as Record<string, unknown>;
        return (
          Array.isArray(obj.obs_mean) &&
          Array.isArray(obj.obs_var) &&
          obj.obs_mean.length === OBSERVATION_SIZE &&
          obj.obs_var.length === OBSERVATION_SIZE &&
          typeof obj.clip_obs === 'number' &&
          typeof obj.epsilon === 'number'
        );
      }

      // ──────────────────────────────────────────────────────────
      // Helpers
      // ──────────────────────────────────────────────────────────

      const clamp = (v: number, lo: number, hi: number): number =>
        Math.max(lo, Math.min(hi, v));

      // ──────────────────────────────────────────────────────────
      // BrowserAIRunner
      // ──────────────────────────────────────────────────────────

      export class BrowserAIRunner {
        private session: ort.InferenceSession | null = null;
        private stats: VecNormStats | null = null;

        /**
         * Load the ONNX model and VecNormalize stats.
         * Must be called once before infer(). Call during loading screen.
         *
         * @param modelUrl - Path to the .onnx model file (e.g., '/assets/model.onnx')
         * @param statsUrl - Path to the VecNormalize stats JSON (e.g., '/assets/vecnorm_stats.json')
         */
        async load(modelUrl: string, statsUrl: string): Promise<void> {
          const [session, statsJson] = await Promise.all([
            ort.InferenceSession.create(modelUrl, {
              executionProviders: ['wasm'],
              graphOptimizationLevel: 'all',
              enableCpuMemArena: true,
              enableMemPattern: true,
            }),
            fetch(statsUrl).then(r => {
              if (!r.ok) throw new Error(`Failed to load stats: ${r.status} ${r.statusText}`);
              return r.json();
            }),
          ]);

          // Runtime validation — the JSON comes from a Python export and may have wrong shape
          if (!isValidStatsJson(statsJson)) {
            throw new Error(
              `Invalid VecNormStats from ${statsUrl}: expected obs_mean[${OBSERVATION_SIZE}], obs_var[${OBSERVATION_SIZE}], clip_obs, epsilon`
            );
          }

          // Remap snake_case JSON keys to camelCase TypeScript interface
          this.stats = {
            obsMean: statsJson.obs_mean,
            obsVar: statsJson.obs_var,
            clipObs: statsJson.clip_obs,
            epsilon: statsJson.epsilon,
          };
          this.session = session;
        }

        /**
         * Run one inference step.
         *
         * @param rawObs - Raw 14-value observation vector from buildObservation()
         * @returns [steer, throttle, brake] — each clamped to its valid range
         */
        async infer(rawObs: number[]): Promise<[number, number, number]> {
          if (!this.session || !this.stats) throw new Error('BrowserAIRunner not loaded');

          const normalizedObs = normalizeObservation(rawObs, this.stats);
          const inputTensor = new ort.Tensor(
            'float32',
            new Float32Array(normalizedObs),
            [1, OBSERVATION_SIZE],
          );

          const results = await this.session.run({ obs: inputTensor });

          const actionsTensor = results['actions'];
          if (!actionsTensor) {
            throw new Error('ONNX model missing "actions" output');
          }
          const actions = actionsTensor.data as Float32Array;

          const output: [number, number, number] = [
            clamp(actions[0], -1, 1),   // steer
            clamp(actions[1], 0, 1),    // throttle
            clamp(actions[2], 0, 1),    // brake
          ];

          // Free WASM-backed output tensors immediately to prevent memory leak
          for (const key of Object.keys(results)) {
            results[key].dispose?.();
          }

          return output;
        }

        /**
         * Release the ONNX session and free WASM memory.
         * Call when leaving the race or switching tracks.
         */
        async dispose(): Promise<void> {
          if (this.session) {
            await this.session.release();
            this.session = null;
            this.stats = null;
          }
        }
      }

  ### Research Insights

  **ONNX Runtime Best Practices (60Hz inference):**
  - `session.run()` in WASM backend is pseudo-async (synchronous computation, Promise wrapper). For a 14->64->64->3 MLP, actual computation is < 0.1ms — well within the 16.7ms frame budget.
  - `enableCpuMemArena: true` and `enableMemPattern: true` eliminate per-run malloc overhead at 60Hz. Memory arena size stabilizes after first few inference calls.
  - `numThreads = 1` is optimal for this tiny model — thread creation/sync overhead exceeds computation time for ~5,500 FLOPs.
  - Output tensors MUST be disposed after extracting values. Without disposal, WASM heap leaks ~43 KB/min (~1.3 MB over 30 min session).

  **TypeScript Type Safety:**
  - `as VecNormStats` on `fetch().json()` is unsafe — the `any` return from `json()` bypasses strict mode entirely. Use a runtime type guard instead.
  - `results['actions'].data as Float32Array` chains two unchecked assertions. Guard with `if (!actionsTensor)` before accessing `.data`.
  - Import `OBSERVATION_SIZE` from `observations.ts` instead of hardcoding `14` — prevents silent shape mismatch if observation vector changes.

  **Vitest Mocking:**
  - `vi.mock()` is hoisted above imports. Mock fixtures declared in outer scope are captured by closure.
  - Arrow functions cannot be constructors in Vitest 4+. Use `vi.fn(function(...) {...})` for `ort.Tensor`.
  - `vi.stubGlobal('fetch', ...)` with `vi.unstubAllGlobals()` in `afterEach` is the correct pattern for mocking `fetch` in a non-DOM test environment.
  - This is the first use of `vi.mock()` in the test suite — existing tests use real implementations. Document the pattern clearly.

  **Performance Considerations:**
  - Pre-allocating a `Float32Array(14)` input buffer and reusing it across ticks would eliminate GC pressure. OPTIONAL — at 60Hz with 14-element arrays, the allocation is trivially cheap compared to the raycaster's ~3,060 intersection tests per tick.
  - Pre-computing `invStdDev[i] = 1 / Math.sqrt(obsVar[i] + epsilon)` at load time replaces per-tick `Math.sqrt`. OPTIONAL micro-optimization.

  **Security:**
  - Model and stats URLs should remain hardcoded paths, not derived from user input. Consider adding a same-origin guard if URLs are ever parameterized.
  - Add `response.ok` check to catch 404s with a clear error message instead of a cryptic `SyntaxError` from trying to parse HTML as JSON.

  **Edge Cases:**
  - If ONNX model was exported without `output_names=["actions", ...]`, output keys are auto-generated numerics (e.g., `'683'`). Validate with `session.outputNames` at load time.
  - If VecNormalize stats have negative variance values, `Math.sqrt(negative)` produces `NaN`. The type guard catches missing/wrong-type fields but not negative variance — acceptable since variance is always non-negative from SB3's RunningMeanStd.

  - [Tensor.dispose() API](https://onnxruntime.ai/docs/api/js/interfaces/Tensor-1.html)

     * AiCarRenderer — AI ghost car renderer (VIS-06).

     * Wraps a CarRenderer with cyan tint, semi-transparent alpha, and a GlowFilter
     * to make the AI car visually distinct from the player car.
     */

    import { CarRenderer } from './CarRenderer';
    import type { Sprite, Graphics } from 'pixi.js';

    // ──────────────────────────────────────────────────────────
    // Visual Constants
    // ──────────────────────────────────────────────────────────

    const AI_TINT         = 0x00eeff;
    const AI_ALPHA        = 0.55;
    const GLOW_DISTANCE   = 8;
    const GLOW_STRENGTH   = 1.5;
    const GLOW_QUALITY    = 0.3;

    // ──────────────────────────────────────────────────────────
    // AiCarRenderer
    // ──────────────────────────────────────────────────────────

    export class AiCarRenderer {
      readonly container: CarRenderer['container'];
      private carRenderer: CarRenderer;

      constructor() {
        this.carRenderer = new CarRenderer();
        this.container = this.carRenderer.container;

        // Apply tint to individual children (Sprite/Graphics have .tint; Container does not)

  ### Research Insights

  **pixi-filters v6 Compatibility:**
  - pixi-filters v6.1.5 is confirmed compatible with PixiJS v8 (`peerDependencies: "pixi.js": ">=8.0.0-0"`)
  - GlowFilter provides both GLSL and WGSL shaders — works on WebGL and WebGPU renderers
  - Historical sharp-edge/padding bug (#217) was fixed in v6 — padding auto-calculated from `distance`
  - Import from `pixi-filters/glow` (tree-shakeable), NOT the deprecated `@pixi/filter-glow` package

  **GlowFilter Performance:**
  - Single GlowFilter on one ~30x60 pixel sprite: negligible GPU cost (+2 draw calls, small texture allocation)
  - PixiJS v8 caches filter render textures between frames — only re-renders when sprite content changes
  - At `quality: 0.3`, the glow uses minimal passes — adequate for a small car sprite
  - If performance issues arise: set `container.filterArea = new Rectangle(...)` to skip bounds measurement

  **PixiJS v8 Container.tint:**
  - `Container` does NOT have a `tint` property in PixiJS v8 — only `Sprite`, `Graphics`, and other display objects do
  - Setting `container.tint` is silently ignored at runtime — the ghost car would appear with default colors
  - Two solutions: (a) iterate children and set tint individually, (b) modify CarRenderer to accept a color scheme
  - Solution (a) is simpler and doesn't require changing CarRenderer's API

  **Design Decision — Delegation vs. Reimplementation:**
  - The simplicity reviewer suggested eliminating AiCarRenderer entirely and inlining 3 lines in WorldRenderer
  - Counter-argument: AiCarRenderer encapsulates the tint-iteration logic and the glow filter setup, which is more than 3 lines with the PixiJS v8 tint workaround
  - Decision: Keep AiCarRenderer but delegate `update()` to CarRenderer instead of reimplementing it
