      Cases for computeGapSeconds (pure logic — no PixiJS):
      - Human arrives at tick 100, AI arrives at tick 160 → gap = (160 - 100) / 60 = +1.0s (player is ahead)
      - Human arrives at tick 160, AI arrives at tick 100 → gap = (100 - 160) / 60 = -1.0s (player is behind)
      - Human and AI arrive at same tick → gap = 0.0s
      - AI has not crossed checkpoint yet (aiArrivalTick = undefined) → gap = 0.0 (edge case — caller handles undefined check)
      - Large difference: human tick 60, AI tick 420 → +6.0s ahead
      - Negative gap sign is correct: player is BEHIND when negative (arrived LATER than AI)

      - showGap(negative) makes container visible, text is '-N.NNs', text fill is red (#ff4466)
      - tick() called 300 times after showGap → container becomes invisible
      - reset() makes container invisible immediately regardless of displayTicksLeft

      ```typescript
      import { Container, Text, Graphics } from 'pixi.js';

      /** Ticks before the gap timer fades out (5 seconds = 300 ticks). */
      const GAP_DISPLAY_TICKS = 300;

      export function computeGapSeconds(humanTick: number, aiArrivalTick: number): number {
        return (aiArrivalTick - humanTick) / 60;
      }

      export class GapTimerHud {
        readonly container: Container;
        private text: Text;
        private bg: Graphics;
        private displayTicksLeft = 0;
        private lastGapSeconds = 0;

        constructor() { ... }  // build text + background panel

        showGap(gapSeconds: number): void {
          this.lastGapSeconds = gapSeconds;
          this.displayTicksLeft = GAP_DISPLAY_TICKS;
          this.updateDisplay();
          this.container.visible = true;
        }

        tick(): void {
          if (this.displayTicksLeft > 0) {
            this.displayTicksLeft--;
            if (this.displayTicksLeft === 0) {
              this.container.visible = false;
            }
          }
        }

        reset(): void {
          this.displayTicksLeft = 0;
          this.container.visible = false;
        }

        private updateDisplay(): void {
          const abs = Math.abs(this.lastGapSeconds);
          const sign = this.lastGapSeconds >= 0 ? '+' : '-';
          this.text.text = `${sign}${abs.toFixed(2)}s`;
          this.text.style.fill = this.lastGapSeconds >= 0 ? '#44ff88' : '#ff4466';
        }
      }
      ```

    <behavior>
      CelebrationOverlay shows the win/loss result when comparing human and AI best laps at race end.
      The comparison logic (computeOutcome) must be exported and unit-tested before the PixiJS overlay is tested visually.

      Cases for computeOutcome (pure logic — no PixiJS):
      - aiTicks = null → 'no-ai-best' (AI has never completed a lap on this track)
      - aiTicks = -1 → 'no-ai-best' (AI timing sentinel value for "no lap completed")
      - aiTicks = 0 → 'no-ai-best' (invalid — AI cannot have completed a lap in 0 ticks)
      - humanTicks <= 0 (human did not complete a lap, e.g. -1) → 'ai-wins' (player did not beat anything)
      - humanTicks > 0, humanTicks < aiTicks → 'human-wins' (player faster)
      - humanTicks > 0, humanTicks > aiTicks → 'ai-wins' (AI faster)
      - humanTicks > 0, humanTicks === aiTicks → 'ai-wins' (tie goes to AI — player must strictly beat)
    </behavior>
    <implementation>
      Export computeOutcome as a standalone pure function from CelebrationOverlay.ts:

      ```typescript
      export type CelebrationOutcome = 'human-wins' | 'ai-wins' | 'no-ai-best';

      /**
       * Determine the race outcome for the celebration overlay.
       * @param humanTicks - Human best lap ticks. Positive = lap completed. <=0 = no lap.
       * @param aiTicks - AI best lap ticks from Leaderboard. null = no AI time set.
       */
      export function computeOutcome(humanTicks: number, aiTicks: number | null): CelebrationOutcome {
        if (aiTicks === null || aiTicks <= 0) return 'no-ai-best';
        if (humanTicks > 0 && humanTicks < aiTicks) return 'human-wins';
        return 'ai-wins';
      }
      ```
