  ### Research Insights — Task 1: GapTimerHud

  **Best Practices (from iRacing, F1, Gran Turismo conventions):**
  - Sign convention `positive = player ahead, negative = player behind` is the universal motorsport standard. Green for ahead, red for behind matches every real and simulated racing HUD.
  - Display with **2 decimal places** (hundredths of a second) — matches `abs.toFixed(2)` already in the plan.
  - Place the gap timer **center-top of screen**, below the countdown area. This is where racing games universally place delta information — visible without moving eyes far from road.

  **Bug Fix — GapTimerHud.tick() Frame-Rate Dependency (GAP-G):**
  The `tick()` method decrements `displayTicksLeft` once per call. If `tick()` is called from the render loop (which runs at monitor refresh rate), the timer fades in 2s at 144Hz instead of 5s at 60Hz.
  - **Fix:** Either (a) call `tick()` from the physics sub-step loop (60Hz fixed), not the render callback, or (b) use elapsed time:
    ```typescript
    tick(deltaTicks: number = 1): void {
      if (this.displayTicksLeft > 0) {
        this.displayTicksLeft -= deltaTicks;
        if (this.displayTicksLeft <= 0) {
          this.displayTicksLeft = 0;
          this.container.visible = false;
        }
      }
    }
    ```
  - **Recommendation:** Call from physics loop. The HudRenderer.render() method should NOT call gapTimerHud.tick(). Instead, GameLoop should call it per physics sub-step or pass physics ticks elapsed.

  **Bug Fix — Duplicate Gap Computation (Pattern Recognition):**
  `computeGapSeconds()` in GapTimerHud.ts and `computeGap()` in GameLoop.ts (Task 3) duplicate the same `(aiTick - humanTick) / 60` formula.
  - **Fix:** GameLoop.computeGap() should import and call `computeGapSeconds()` from GapTimerHud.ts instead of reimplementing it. Single source of truth.
  - **Alternative (stronger):** Extract to `src/utils/race-math.ts` as a pure utility, import from both. This avoids a renderer → renderer import and keeps the function in a utils layer.

  **Performance Considerations:**
  - computeGapSeconds is trivially cheap (one subtraction, one division). No concerns.
  - GapTimerHud PixiJS objects (Text, Graphics) are allocated once in constructor and reused. No per-frame allocations.

  **Edge Cases:**
  - Gap of exactly 0.0s: Display "+0.00s" in green (player matched AI exactly — treat as positive).
  - Gap timer should only be active in `vs-ai` mode, NOT spectator (CP-9). The plan correctly states "Only visible in vs-ai mode" but the check should be `mode === 'vs-ai'`, not `mode !== 'solo'`.
  - On multi-lap races, the gap at checkpoint N in lap 2 should compare lap 2 arrivals, not lap 1 (CP-8). This is enforced in Task 3's checkpoint tracking logic.

  **Test File Location:**
  Pattern Recognition agent flagged that `tests/ai/gap-timer.test.ts` is in `tests/ai/` but tests code from `src/renderer/`. Since the test only exercises the pure function (no PixiJS), this is acceptable, but `tests/renderer/gap-timer.test.ts` would be more conventional. Keep current location to avoid churn.
