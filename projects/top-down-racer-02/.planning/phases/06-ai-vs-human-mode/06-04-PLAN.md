    ```typescript
    private checkBestTime(): void {
      const trackId = TRACKS[this.activeTrackIndex].id;

      // Human best (all modes — in spectator, human never completes a lap so this no-ops)
      const timing = this.gameLoop.currentWorldState.timing;
      if (timing.bestLapTicks > 0 && timing.bestLapTicks !== this.lastBestLapTicks) {
        setHumanBest(trackId, timing.bestLapTicks);
        this.lastBestLapTicks = timing.bestLapTicks;
      }

      // AI best (vs-ai and spectator modes — solo has no AI world)
      if (this.currentMode !== 'solo') {
        const aiBest = this.gameLoop.currentAiWorldState?.timing.bestLapTicks;
        if (aiBest != null && aiBest > 0 && aiBest !== this.lastAiBestLapTicks) {
          setAiBest(trackId, aiBest);
          this.lastAiBestLapTicks = aiBest;
        }
      }
    }
    ```

### Performance Notes

- **localStorage reads during build():** 3 calls to `getLeaderboard()` = 3x `getItem` + `JSON.parse`. Total cost <0.01ms. No caching needed at 3 tracks.
- **Text objects per card:** Adding 1 Text (AI time) per card = 3 extra Text objects total. Negligible GPU cost.
- **Destroy/rebuild pattern:** Acceptable for TrackSelectScreen (refreshes once per race-to-menu transition, not per-frame). Cost ~2-3ms for 3 cards.
- **BitmapText consideration:** For HUD elements that update every frame (gap timer, lap times), BitmapText is more efficient than Canvas Text. Not required for TrackSelectScreen's static text.

### Security Notes

- localStorage I/O is safe: try/catch on all reads/writes, `Number.isFinite` + `> 0` input validation on ticks.
- PixiJS Text renders to canvas bitmap — zero XSS/injection risk (not DOM-based).
- Consider hardening `getLeaderboard()` return with `typeof entry?.human === 'number'` instead of `?? null` to reject non-number values from tampered localStorage.
