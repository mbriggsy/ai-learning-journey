# Top-Down Racer v03

> *v02 proved the architecture. v03 proves the graphics pipeline and genuine AI generalization.*

A visual and AI upgrade of Top-Down Racer v02 — built entirely by autonomous AI development. Zero lines of hand-written game code.

**The question is no longer whether AI can write disciplined code.  
The question is whether it can make something *beautiful*.**

## What This Is

v03 is a commercial-quality visual upgrade + track evolution of v02. The battle-tested simulation engine is frozen. The AI gets a genuine challenge: redesigned tracks that make memorization statistically impossible, forcing real generalization.

Two goals. One project:
1. **Make it look like a real game** — AI-generated assets, post-processing shaders, commercial HUD, cinematic menus
2. **Make the AI actually learn** — new circuits with no repeated corner shapes; a v02 model that confidently drives into a wall proves the redesign worked

## Getting Started

### Prerequisites

```powershell
# Node.js v20+, pnpm 10.x
node --version
pnpm --version

# Python 3.12 (for AI training)
python --version
```

### Setup

```powershell
# Clone and navigate
cd C:\Users\brigg\ai-learning-journey\projects\top-down-racer-03

# Install dependencies (once scaffold is generated by GSD Phase 1)
pnpm install

# Python AI training environment
cd python
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### Running the Game (post-build)

```powershell
pnpm dev        # Dev server (hot reload)
pnpm build      # Production build
pnpm test       # Run all tests
```

### AI Training

```powershell
cd python
.venv\Scripts\Activate.ps1

# Sanity run (Track 1, ~100K steps)
python train.py --track 1 --timesteps 100000

# Production training (Track 3, ~2M steps)
python train.py --track 3 --timesteps 2000000

# Cross-track validation (inference only — no training)
python validate.py --model models/v03_track3.zip --track 2
```

## GSD Autopilot Usage

v03 introduces **GSD Autopilot** — a meta-orchestrator that chains GSD phases autonomously, handles human gates with skip-ahead logic, and tracks state on disk so it can resume after any interruption.

**Starting a new build:**
```
/gsd-autopilot start
```

**Checking status:**
```
/gsd-autopilot status
```

**Resuming after a human gate (e.g., you've generated assets and dropped them in `/assets/raw/`):**
```
/gsd-autopilot resume
```

**Human gates** are explicit pause points where the autopilot needs something only you can do:
- `external-action` — generate assets with Nano Banana / Ludo.ai / Stitch
- `approval` — review track geometry before AI training begins
- `quality-check` — play the game, approve visual quality
- `decision` — architectural choice not locked in the spec

When blocked, the autopilot keeps building everything it *can* while waiting for you. See `.planning/HUMAN-GATES.md` for clear instructions on what's needed.
